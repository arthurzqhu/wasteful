# WASTEFUL

**W**asteful **A**daptive **S**ubsampling, **T**radeoffs of **E**xercising **F**r**U**ga**L**ity (in PPE-inference procedures)

A toy framework for evaluating training data sampling strategies in emulator-guided Bayesian parameter inference. The core question: when simulator evaluations are expensive (e.g., Earth system models), how should you allocate your limited budget of Perturbed Parameter Ensemble (PPE) runs across iterative refinement rounds?

## Motivation

In climate and Earth system modeling, running a single PPE can take hours to days on HPC clusters. Practitioners typically afford one large initial ensemble, then must be frugal in subsequent rounds as they refine parameter estimates. This creates a fundamental tension:

- **More data coverage** improves emulator accuracy across the full parameter space
- **Focused data** near the current posterior estimate improves accuracy where it matters most
- **Reusing historical data** from prior rounds avoids waste but may include irrelevant regions

WASTEFUL uses cheap synthetic simulators to study these tradeoffs in a controlled setting where the ground truth is known.

## Three Strategies

Each round, a PPE is generated by sampling parameters from the current posterior. The strategies differ in how they select training data for the emulator:

| Strategy | Training data | Bounds update |
|---|---|---|
| **Global** | All PPEs from all rounds | None (original prior) |
| **Buffered** | All PPEs within current bounds + margin | 3-sigma + configurable buffer margin around posterior |
| **Local** | All PPEs from all rounds that fall within current bounds | Strict min/max of previous posterior |

## Emulators

Two emulator architectures are compared:

- **Gaussian Process** (GP): tinygp with ARD squared-exponential kernel. Multi-restart optimization (5 restarts) with hyperparameter clamping to avoid degenerate solutions.
- **Neural Network** (NN): Flax MLP trained with CRPS loss for calibrated uncertainty. Hyperparameter search over layer counts, widths, and learning rates with early stopping.

Both produce predictions with uncertainty estimates, which are used in the MCMC likelihood in place of the true simulator.

## How to Run

### Dependencies

Only tested with Python 3.13.

```bash
pip install jax jaxopt flax optax tinygp blackjax matplotlib corner numpy ipython
```

### Single parameterization (development/debugging)

```bash
python main_experiment.py harmonic_mean
python main_experiment.py power_law
```

Runs one parameterization through the full pipeline: discovery round, iterative refinement, plotting. A parameterization name is required. Available options: `power_law`, `harmonic_mean`, `harmonic_mean_3t`, `hill_growth`.

### Multiple parameterizations (full comparison)

```bash
python run_all_parameterizations.py                          # all parameterizations
python run_all_parameterizations.py power_law harmonic_mean  # subset
```

Thin wrapper around `main_experiment.run_experiment()` that loops over parameterizations and prints a consolidated best-of summary at the end.

### Buffer sensitivity study

```bash
python buffer_sensitivity_study.py                  # all parameterizations
python buffer_sensitivity_study.py harmonic_mean    # single parameterization
```

Sweeps the buffer margin parameter (0% to 200% in 10 steps) to characterize its effect on convergence for the buffered strategy. Automatically scales `n_ppe_r1` and refinement rounds to the parameterization's dimensionality.

By default uses GP emulators. To use NN instead, change `MODEL_TYPE = "nn"` at the top of the file.

Outputs per parameterization:
- `plots/{name}_buffer_sensitivity.pdf` — convergence trajectories and final RMSE bar chart
- `plots/{name}_buffer_posteriors.pdf` — per-parameter posterior distributions across buffer scales
- `results/{name}_buffer_sensitivity.csv` — per-round metrics for each buffer scale

## How to Modify

### Adding a new simulator function

Define a function in `main_experiment.py` with signature `f(theta, x)` where `theta` is `(n_ppe, n_params)` and `x` is `(n_obs,)`. Then add it to the `PARAMETERIZATIONS` dict in `main_experiment.py`:

```python
"my_model": {
    "sim_func": my_model_func,
    "true_theta": jnp.array([...]),              # ground truth parameters
    "theta_bounds": [(lo, hi), ...],             # one tuple per parameter
    "x_bounds": (x_lo, x_hi),                   # independent variable range
},
```

The number of refinement rounds is automatically set to the number of parameters.

### Tuning the compute budget

Key knobs in `main_experiment.py`:

| Parameter | Location | Effect |
|---|---|---|
| `n_ppe_r1` | `main_experiment.py`: `50 * n_params²` | Discovery round PPE size. Scales quadratically with dimensionality. Set to `None` for auto-detection. |
| `n_ppe_r2` | `main_experiment.py`: `n_ppe_r1 // 10` | Refinement round PPE size. Small by design to mimic expensive simulators. Set to `None` for auto-detection. |
| `n_refinement_rounds` | `main_experiment.py`: `n_params` | Number of refinement rounds after discovery. Set to `None` for auto-detection. |
| `n_mcmc_samples` | `main_experiment.py` | MCMC steps per round (post-warmup samples = steps/2) |
| `UNCERTAINTY_MULTIPLIER` | `main_experiment.py` | Inflates emulator uncertainty in MCMC to prevent overconfident posteriors |
| `buffer_margin` | `IterativeSolution.__init__` | Fractional padding for the buffered strategy (default 0.2 = 20% of prior width) |

### Adding a new strategy

Extend `IterativeSolution.update_priors()` to define how bounds shrink, and `IterativeSolution.train()` to define which historical PPEs are included in training data.

### Adding a new emulator

Implement `train_*` and `predict_*` functions in `emulators.py` (returning mean and std), add a corresponding `get_*_logprob` in `mcmc_sampler.py`, and wire them into `IterativeSolution.train()` and `IterativeSolution.get_logprob_fn()`.

## Architecture

```
simulator.py          # Simulator interface (true_process evaluation)
emulators.py          # GP and NN emulator implementations
mcmc_sampler.py       # BlackJAX NUTS sampler + log-probability functions
main_experiment.py    # Core pipeline, IterativeSolution class, shared output routines, single-run entry point
run_all_parameterizations.py   # Multi-parameterization runner (thin wrapper around main_experiment)
buffer_sensitivity_study.py    # Buffer margin sweep
```

**Pipeline per strategy:**
1. **Discovery** (Round 1): Generate large PPE with uniform sampling across full prior bounds. Train emulator. Run MCMC to get initial posterior.
2. **Refinement** (Rounds 2+): Sample new PPE from previous posterior. Select training data per strategy. Retrain emulator. Run MCMC with tighter uncertainty scaling.
3. **Analysis**: Corner plots overlaying all rounds, normalized parameter distance and simulator RMSE tracked per round.

## Output

| Directory | Contents |
|---|---|
| `posteriors/` | `.npz` files with MCMC samples per round/strategy |
| `plots/` | PDF corner plots per strategy, emulator performance (MAS) parity plots per round, overlaying rounds with viridis colormap |
| `results/` | CSV files: `{name}_metrics.csv` (per-round normalized distance, RMSE, n_train) and `{name}_timing.csv` |

## Assumptions and Caveats

### Simplifications relative to real ESMs

- **Cheap simulator**: The toy simulators evaluate instantly. Real ESMs take hours per evaluation, which means the cost of generating PPEs dominates total runtime. The current timing metrics reflect emulator training and MCMC cost only.
- **Low dimensionality**: 2-4 parameters. Real ESMs may have 10-50+ tunable parameters, which changes the curse-of-dimensionality calculus significantly (GP scaling, required PPE sizes, MCMC mixing).
- **Smooth, well-behaved functions**: The test functions are continuous and differentiable. Real ESMs can have discontinuities, regime transitions, and chaotic sensitivity to parameters.
- **Known ground truth**: We know the true parameters and can measure convergence directly. In practice, you only have observations with uncertain error models.
- **Scalar output**: Each simulator call produces one output per x-value. Real ESMs produce spatiotemporal fields requiring dimensionality reduction before emulation.

### Known limitations

- **GP scaling**: GP training is O(n^3) in the number of training points. The global strategy accumulates all data, which becomes slow for many rounds or large PPEs.
- **NN uncertainty calibration**: The NN uses CRPS loss for uncertainty, which is more robust than NLL but still relies on the Gaussian assumption for the predictive distribution.
- **Uncertainty multiplier**: The `UNCERTAINTY_MULTIPLIER` is a manual knob that prevents the posterior from collapsing due to overconfident emulators. Its optimal value is problem-dependent and not automatically tuned.
- **Local strategy fragility**: The local strategy uses strict min/max of the previous posterior as bounds. If a single round produces a biased posterior (common with emulator-guided MCMC), the bounds permanently exclude the true parameter values and the strategy cannot recover. This manifests as rectangular/uniform posteriors in later rounds — the emulator has too few training points in the narrow bounds to learn meaningful parameter-output relationships, so the GP lengthscales hit their upper clamp limits, the likelihood becomes flat, and the MCMC samples the prior uniformly. This is by design as a demonstration of aggressive bound-shrinking risk; the buffered strategy mitigates this with wider margins.
- **Parameter identifiability**: Some test functions have parameter degeneracies (e.g., the harmonic mean functions can show bias in certain parameters depending on the x range). This is realistic but means not all strategies will converge to truth.
- **Single random seed**: Results can vary with different random seeds. For publication-quality comparisons, average over multiple seeds.
